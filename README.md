# NetChat

Консольный **peer-to-peer** чат с прямыми соединениями (без центрального сервера). На старте указывается имя пользователя и, опционально, адрес peer-а для подключения; если адрес не указан — приложение работает в режиме ожидания входящих подключений.

Проект учебный: важны аккуратная документация, тестирование и CI.

## Технологии (план)

- **Язык:** Java 21 (LTS)
- **Сборка:** Maven
- **Транспорт:** TCP (прямые соединения)
- **Протокол сообщений:** NDJSON (JSON-объекты, разделённые `\n`), UTF-8
- **Тесты:** JUnit 5
- **CI:** GitHub Actions (сборка, тесты, проверка форматирования)

gRPC рассматривается как опциональное расширение, но базовая версия намеренно остаётся простой (без «архитектурной жадности»).

## Сборка и запуск (после реализации)

Зависимости:

- JDK 21+
- Maven 3.9+

Сборка:

```bash
mvn -B -ntp verify
```

Примеры целевого CLI (зафиксированы в требованиях и будут совпадать с реализацией):

- **Ожидание подключения (server mode):** `java -jar netchat.jar --name Alice --listen 0.0.0.0:9000`
- **Подключение к peer (client mode):** `java -jar netchat.jar --name Bob --peer 192.168.1.5:9000`
